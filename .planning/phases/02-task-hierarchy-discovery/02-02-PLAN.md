**1. HierarchyConstructionStage** (`hierarchy_construction.py`):

Create `HierarchyConstructionStage(DiscoveryStage)` with `name = "hierarchy_construction"`.

In `execute(self, job: DiscoveryJob) -> DiscoveryJob`:
- Convert `job.task_candidates` into `DiscoveredTask` objects:
  - For each candidate, create a `DiscoveredTask` with fields copied from the candidate.
  - Build parent-child relationships using FOLIO concept hierarchy (via FolioService).
  - For heading-based candidates, use the heading_path hierarchy: if heading_path is `["Ch.5", "Cross-Examination", "Leading Questions"]`, then "Leading Questions" is a child of "Cross-Examination", which is a child of "Ch.5".
  - Set `parent_task_id` for each task based on the nearest matching parent task.
  - Set `depth` based on tree level.
- Handle FOLIO polyhierarchy: if a task maps to a FOLIO concept with multiple `parent_iris`, store all parent IRIs in `task.parent_iris`. The task appears once in the flat task list but can be shown under multiple parents in the tree view.
- Determine task types using LLM (via `LLMBridge.get_llm_for_task("task_ordering")`):
  - For each top-level task, ask LLM if it's procedural (ordered steps) or categorical (collection of techniques).
  - If procedural, ask LLM for canonical ordering of subtasks (preparation -> execution -> follow-up pattern). Set `canonical_order` on each child.
  - Fallback: if LLM fails, default to `is_procedural = False` (categorical).
- Build `task_unit_links` and `unit_task_links` dictionaries:
  - For each task, link all `knowledge_unit_ids` from its TaskCandidate.
  - For each unit, record all tasks it's linked to.
  - Compute `unit_type_counts` per task by counting unit types.
- Assign orphan units (units not linked to any task):
  - Find the nearest task by embedding similarity (reuse SentenceTransformer from task_clustering).
  - Auto-assign to nearest task, add unit ID to `job.orphan_unit_ids` for flagging.
- Group knowledge units by type under each task: count how many of each `KnowledgeType` (advice, principle, citation, rule, pitfall) are linked to each task. Store in `task.unit_type_counts`.
- Detect jurisdiction sensitivity: for each unit, check if its text mentions jurisdictional variation using the `JURISDICTION_DETECTION_PROMPT` from task_discovery.py prompts (or a simpler heuristic: regex for "in federal court", "some states", "jurisdictional", "varies by state/jurisdiction"). Set `task.is_jurisdiction_sensitive = True` if any linked unit is flagged.
- Set `job.discovered_tasks` and `job.task_hierarchy = TaskHierarchy(tasks=..., task_unit_links=..., unit_task_links=..., orphan_unit_ids=...)`.

**2. CrossSourceMergingStage** (`cross_source_merging.py`):

Create `CrossSourceMergingStage(DiscoveryStage)` with `name = "cross_source_merging"`.

In `execute(self, job: DiscoveryJob) -> DiscoveryJob`:
- Two merge strategies:
  1. **FOLIO IRI matching (exact):** Tasks with the same `folio_iri` from different source files are merge candidates. Merge by keeping the task with highest aggregate confidence as canonical.
  2. **Embedding similarity (fuzzy):** For tasks without FOLIO IRI match, encode task labels using `SentenceTransformer("all-MiniLM-L6-v2")` and find pairs with cosine similarity > 0.85 (same threshold as Phase 1 dedup). Merge these.
- Merge procedure:
  - Keep the canonical task (highest confidence).
  - Move all `task_unit_links` from the merged task to the canonical task.
  - Update `unit_task_links` accordingly.
  - Remove the merged task from `job.task_hierarchy.tasks`.
  - If merged tasks had different parents, use the canonical task's parent.
  - Log: "Merged '{merged_label}' into '{canonical_label}' (reason: {iri_match|embedding_similarity})".
- After merging, recompute `unit_type_counts` for all affected tasks.
- Update `job.task_hierarchy` with the merged result.

**3. ContradictionDetectionStage** (`contradiction_detection.py`):

Create `ContradictionDetectionStage(DiscoveryStage)` with `name = "contradiction_detection"`.

In `execute(self, job: DiscoveryJob) -> DiscoveryJob`:
- For each task in `job.task_hierarchy.tasks`:
  - Get all linked knowledge unit IDs from `task_unit_links`.
  - Look up actual KnowledgeUnit objects from `job.knowledge_units`.
  - If a task has fewer than 2 units, skip (no pairs to compare).
  - Call `ContradictionDetector.screen_pairs(units)` for NLI screening.
  - For each pair above threshold (0.7), call `ContradictionDetector.deep_analyze(unit_a, unit_b)`.
  - If LLM confirms contradiction, create a `Contradiction` object and append to `job.contradictions`.
  - Set `task.has_contradictions = True` for affected tasks.
- Update `job.task_hierarchy.contradictions` with all found contradictions.

```python
class TaskDiscoveryOrchestrator:
    """Orchestrate the 6-stage task discovery pipeline.

    Reads Phase 1 extraction output and discovers advocacy tasks
    organized in a hierarchical tree. Supports checkpoint-based
    resume for interrupted runs. Loads previously approved task
    decisions from SQLite so reviewer edits survive re-runs.
    """

    def __init__(self, settings: Settings, db_path: Path | None = None) -> None:
        self.settings = settings
        self._db_path = db_path  # Path to SQLite DB; None = no decision persistence
        self._stages: list[DiscoveryStage] = self._build_stages()

    def _build_stages(self) -> list[DiscoveryStage]:
        # Lazy imports like PipelineOrchestrator
        from folio_insights.pipeline.discovery.stages.heading_analysis import HeadingAnalysisStage
        from folio_insights.pipeline.discovery.stages.folio_mapping import FolioMappingStage
        from folio_insights.pipeline.discovery.stages.content_clustering import ContentClusteringStage
        from folio_insights.pipeline.discovery.stages.hierarchy_construction import HierarchyConstructionStage
        from folio_insights.pipeline.discovery.stages.cross_source_merging import CrossSourceMergingStage
        from folio_insights.pipeline.discovery.stages.contradiction_detection import ContradictionDetectionStage

        return [
            HeadingAnalysisStage(),
            FolioMappingStage(),
            ContentClusteringStage(),
            HierarchyConstructionStage(),
            CrossSourceMergingStage(),
            ContradictionDetectionStage(),
        ]

    async def _load_approved_decisions(self, corpus_name: str) -> list[DiscoveredTask]:
        """Load previously approved task_decisions from SQLite.

        Returns a list of DiscoveredTask objects reconstructed from the
        task_decisions table where status='approved' or status='edited'.
        These are injected into the DiscoveryJob so that subsequent
        pipeline stages treat them as locked (not overwritten).

        If db_path is None or the table doesn't exist yet (first run),
        returns an empty list.
        """
        if self._db_path is None or not self._db_path.exists():
            return []
        import aiosqlite
        approved: list[DiscoveredTask] = []
        async with aiosqlite.connect(self._db_path) as db:
            db.row_factory = aiosqlite.Row
            try:
                async with db.execute(
                    "SELECT * FROM task_decisions WHERE corpus_name = ? AND status IN ('approved', 'edited')",
                    (corpus_name,),
                ) as cursor:
                    async for row in cursor:
                        task = DiscoveredTask(
                            id=row["task_id"],
                            label=row.get("edited_label") or row["label"],
                            folio_iri=row.get("folio_iri"),
                            parent_task_id=row.get("parent_task_id"),
                            is_procedural=bool(row.get("is_procedural", 0)),
                            canonical_order=row.get("canonical_order"),
                            is_manual=bool(row.get("is_manual", 0)),
                            review_status=row["status"],
                        )
                        approved.append(task)
            except Exception:
                # Table may not exist on first run -- return empty
                pass
        return approved

    async def run(self, corpus_name: str, resume: bool = True) -> DiscoveryJob:
        """Load Phase 1 extraction output and run all 6 discovery stages.

        CRITICAL: Before running stages, loads approved task_decisions from
        SQLite (if any exist) and injects them into the DiscoveryJob. Stages
        must treat these as locked -- they are not re-discovered or overwritten.
        After all stages complete, computes a diff between new discovery results
        and the previously approved state for the frontend DiffView.
        """
        # 1. Load extraction.json from Phase 1 output
        corpus_dir = self.settings.output_dir / corpus_name
        extraction_path = corpus_dir / "extraction.json"
        # Parse extraction data, build list of KnowledgeUnit objects

        # 2. DECISION PERSISTENCE: Load approved decisions from SQLite
        approved_tasks = await self._load_approved_decisions(corpus_name)
        # Store the pre-run approved state for diff computation later
        pre_run_task_ids = {t.id for t in approved_tasks}
        pre_run_tasks = {t.id: t for t in approved_tasks}

        # 3. Create DiscoveryJob, injecting approved tasks as locked
        # Set job.metadata["locked_task_ids"] = list of approved task IDs
        # Set job.discovered_tasks = approved_tasks (pre-populated)
        # Stages MUST check job.metadata["locked_task_ids"] and skip those tasks

        # 4. Iterate stages with checkpoint-based resume (same pattern as PipelineOrchestrator)
        #    Use DiscoveryCheckpoint (or adapt PipelineCheckpoint for DiscoveryJob serialization)

        # 5. DIFF COMPUTATION: After all stages, compute diff
        #    diff = self._compute_diff(job, pre_run_tasks)
        #    Write diff to {corpus_dir}/discovery_diff.json

        # 6. Write discovery output:
        #    - {corpus_dir}/discovery.json (task hierarchy + contradictions)
        #    - {corpus_dir}/task_tree.json (tree structure for viewer)
        #    - {corpus_dir}/discovery_diff.json (diff for DiffView)
        # 7. Return completed DiscoveryJob

    def _compute_diff(
        self, job: DiscoveryJob, pre_run_tasks: dict[str, DiscoveredTask]
    ) -> list[dict]:
        """Compare new discovery results against previously approved state.

        Returns a list of diff entries for the frontend DiffView component:
        [
            {"type": "added", "id": "...", "description": "New task: Cross-Examination Techniques"},
            {"type": "removed", "id": "...", "description": "Task removed: Old Task Name"},
            {"type": "changed", "id": "...", "description": "Label changed: 'Old' -> 'New'"},
            {"type": "changed", "id": "...", "description": "Parent changed for: Task Name"},
            {"type": "changed", "id": "...", "description": "3 units added to: Task Name"},
        ]

        Comparison logic:
        - For each task in job.task_hierarchy.tasks:
          - If task.id not in pre_run_tasks: type="added"
          - If task.id in pre_run_tasks but label/parent/folio_iri changed: type="changed"
        - For each task_id in pre_run_tasks:
          - If task_id not in current tasks: type="removed"
        - Unit link changes: compare task_unit_links counts per task
        """
        current_tasks = {t.id: t for t in (job.task_hierarchy.tasks if job.task_hierarchy else [])}
        diff: list[dict] = []

        # Added tasks
        for tid, task in current_tasks.items():
            if tid not in pre_run_tasks:
                diff.append({"type": "added", "id": tid, "description": f"New task: {task.label}"})

        # Removed tasks
        for tid, old_task in pre_run_tasks.items():
            if tid not in current_tasks:
                diff.append({"type": "removed", "id": tid, "description": f"Task removed: {old_task.label}"})

        # Changed tasks
        for tid in current_tasks.keys() & pre_run_tasks.keys():
            new_t = current_tasks[tid]
            old_t = pre_run_tasks[tid]
            if new_t.label != old_t.label:
                diff.append({"type": "changed", "id": tid, "description": f"Label changed: '{old_t.label}' -> '{new_t.label}'"})
            if new_t.parent_task_id != old_t.parent_task_id:
                diff.append({"type": "changed", "id": tid, "description": f"Parent changed for: {new_t.label}"})
            if new_t.folio_iri != old_t.folio_iri:
                diff.append({"type": "changed", "id": tid, "description": f"FOLIO mapping changed for: {new_t.label}"})

        return diff
```
