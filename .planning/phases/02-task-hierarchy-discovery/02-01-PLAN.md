**1. HeadingAnalysisStage** (`heading_analysis.py`):

Create a `HeadingAnalysisStage(DiscoveryStage)` with `name = "heading_analysis"`.

In `execute(self, job: DiscoveryJob) -> DiscoveryJob`:
- Group knowledge units by `source_section` (the heading hierarchy path stored on each unit).
- For each unique heading path, create a `TaskCandidate` with:
  - `label` = the deepest heading in the path (e.g., `source_section[-1]`)
  - `source_signal = "heading"`
  - `heading_path = unit.source_section`
  - `knowledge_unit_ids` = IDs of all units under that heading
  - `confidence` = 1.0 for top-level headings, 0.7 for sub-headings, 0.4 for deep headings (same proximity weights as HeadingContextExtractor)
  - `source_file = unit.source_file`
- Filter out heading paths with fewer than 2 knowledge units (too sparse to form a task).
- Append to `job.task_candidates`.
- Log the number of heading-based candidates discovered.

**2. FolioMappingStage** (`folio_mapping.py`):

Create a `FolioMappingStage(DiscoveryStage)` with `name = "folio_mapping"`.

In `execute(self, job: DiscoveryJob) -> DiscoveryJob`:
- For each `TaskCandidate` in `job.task_candidates`:
  - Collect all FOLIO tags from associated knowledge units (using `knowledge_unit_ids` to look up units in `job.knowledge_units`).
  - Find the most frequent FOLIO IRI among those units' `folio_tags`.
  - Use `FolioService` (via bridge) to traverse the hierarchy and find the deepest appropriate concept (per CONTEXT.md: "as deep as possible, but no deeper").
  - Set `candidate.folio_iri` and `candidate.folio_label`.
  - Recompute `candidate.confidence` using `compute_task_confidence(folio_confidence, heading_confidence)` where `folio_confidence` is the average confidence of matching FOLIO tags and `heading_confidence` is the candidate's existing confidence.
- If a FOLIO concept cannot be found for a candidate, mark it for potential new sibling proposal (set `candidate.metadata["proposed_sibling"] = True` via a dict on TaskCandidate or DiscoveryJob metadata).
- Log mapping statistics: N candidates mapped, N proposed new concepts.

**3. ContentClusteringStage** (`content_clustering.py`):

Create a `ContentClusteringStage(DiscoveryStage)` with `name = "content_clustering"`.

In `execute(self, job: DiscoveryJob) -> DiscoveryJob`:
- Call `cluster_units_for_task_discovery(job.knowledge_units, distance_threshold=0.5)` from `task_clustering.py`.
- For each cluster with >= 3 units:
  - Check if a heading-based TaskCandidate already covers most (>70%) of the cluster's units. If so, skip (don't create a duplicate).
  - Otherwise, create a new `TaskCandidate` with `source_signal = "clustering"`, `knowledge_unit_ids` = unit IDs in the cluster.
  - Use LLM (via `LLMBridge.get_llm_for_task("task_discovery")`) with the `TASK_DISCOVERY_PROMPT` to determine: `label`, `is_procedural`, `confidence`.
  - Set `candidate.label` from LLM response.
- Append new clustering-based candidates to `job.task_candidates`.
- Log: N clusters found, N new implicit tasks discovered.

**4. task_clustering.py** (`src/folio_insights/services/task_clustering.py`):

Create `cluster_units_for_task_discovery(units: list[KnowledgeUnit], distance_threshold: float = 0.5) -> list[list[int]]`:
- Use `SentenceTransformer("all-MiniLM-L6-v2")` to encode `[u.text for u in units]` with `normalize_embeddings=True`.
- Use `sklearn.cluster.AgglomerativeClustering` with `n_clusters=None`, `distance_threshold=distance_threshold`, `metric="cosine"`, `linkage="average"`.
- Call `fit_predict(embeddings)` and group unit indices by cluster label.
- Return `list[list[int]]` -- each inner list is indices into the input `units` list.
- Handle edge case: if len(units) < 2, return a single cluster with all indices.

**5. Extend LLMBridge** (`llm_bridge.py`):
- Add `"task_discovery"`, `"task_ordering"`, `"contradiction"`, `"orphan_assignment"` to the `INSIGHTS_TASKS` tuple.
