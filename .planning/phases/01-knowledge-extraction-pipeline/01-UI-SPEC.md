# Phase 1 — UI Design Contract

> Visual and interaction contract for the folio-insights review viewer. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (SvelteKit -- not in shadcn ecosystem) |
| Preset | not applicable |
| Component library | none (hand-built Svelte components) |
| Icon library | Inline SVG (matching folio-mapper pattern -- no external icon library) |

**Source:** folio-enrich and folio-mapper both use system fonts and inline SVG icons. CONTEXT.md requires matching their look/feel. SvelteKit does not have shadcn support; components are hand-built following ecosystem patterns.

---

## Theme Direction

The viewer uses a **dark theme** matching folio-enrich's review interface, since both are data-dense annotation/review tools. folio-mapper's light theme serves a different UX mode (guided wizard). The review viewer's three-pane layout with dense text, confidence scores, and extended reading sessions benefits from the low-glare dark palette.

All CSS custom properties below are sourced from folio-enrich's established tokens with adjustments for the folio-insights review context.

**Primary focal point:** The knowledge unit list in the DetailView upper-right pane is the main visual anchor and review workspace. All layout proportions, contrast, and keyboard shortcuts funnel attention here.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline badge padding, tree indent per level |
| sm | 8px | Compact element spacing, list item vertical gap, chip padding |
| md | 16px | Default element spacing, panel internal padding, form field gaps |
| lg | 24px | Section padding within panes, header horizontal padding |
| xl | 32px | Pane-to-pane gutter (resizable divider zone) |
| 2xl | 48px | Header height (matches folio-enrich's 48px header) |
| 3xl | 64px | Not used in Phase 1 |

Exceptions:
- Tree node row height: 28px (for dense scanning, matching folio-enrich tree rows)
- Review action buttons: minimum 36px height (sufficient touch target for mouse-driven desktop tool -- not a mobile app)

**Source:** folio-enrich header is 48px. Tree rows use compact spacing for dense data scanning. 8-point grid base.

## Typography

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 14px | 400 (regular) | 1.5 | Knowledge unit text, source context, descriptions |
| Label | 11px | 400 (regular) | 1.3 | Status chips, confidence labels, extraction path tags, metadata |
| Heading | 18px | 600 (semibold) | 1.2 | App title, pane headers, section titles, concept headers |
| Tree node | 13px | 400 (regular) | 1.3 | FOLIO concept tree labels, list view items |

**Source:** folio-enrich uses 18px/600 for header title, 11px for status chips, 13px for tree labels. Body at 14px matches the data-dense review context (not a marketing page). Label weight collapsed from 500 to 400; `--text-dim` color provides sufficient visual separation for metadata without a third weight.

**Letter spacing:** -0.3px on heading role only (matching folio-enrich header h1).

## Layout Contract

```
+--------------------------------------------------+
|  Header (48px, --surface, border-b --border)     |
+----------+---------------------------------------+
|          |  Upper-right: FOLIO Detail View        |

|  FOLIO   +---------------------------------------+
|  Tree    |  Lower-right: Source Context           |
|          |  (source text for validation)          |
+----------+---------------------------------------+
```

| Pane | Default Width/Height | Min | Resizable | Content |
|------|---------------------|-----|-----------|---------|
| Left (tree) | 320px | 240px | Yes (horizontal) | FOLIO concept tree filtered to tagged concepts |

| Lower-right (source) | 40% of right area height | 120px | Yes (vertical) | Source text context for "trust but verify" |

**Resizable dividers:** 4px wide/tall drag handles with `--border` color, `--accent` on hover. Cursor: `col-resize` (horizontal), `row-resize` (vertical).

## Interaction Contract

### Keyboard Shortcuts (from CONTEXT.md: "keyboard shortcuts, skim-approve")

| Key | Action | Scope |
|-----|--------|-------|
| `A` | Approve selected unit | Detail pane focused |
| `R` | Reject selected unit | Detail pane focused |
| `E` | Edit selected unit (open inline editor) | Detail pane focused |
| `S` | Skip (move to next without decision) | Detail pane focused |
| `Shift+A` | Approve all high-confidence units in current concept | Any |
| `J` / `ArrowDown` | Next unit in list | Detail pane |
| `K` / `ArrowUp` | Previous unit in list | Detail pane |
| `Enter` | Expand/select tree node | Tree pane focused |
| `Ctrl+F` | Focus tree filter input | Any |
| `Ctrl+Enter` | Save inline edit | Editor open |
| `Escape` | Close editor / dismiss modal | Any |
| `?` | Show keyboard shortcuts overlay | Any |
| `1` / `2` / `3` / `4` | Switch confidence filter (All/High/Medium/Low) | Any |
| `Tab` | Cycle focus between panes (Left -> Upper-Right -> Lower-Right) | Any |

### Review Workflow States

| State | Visual Indicator | Persistence |
|-------|-----------------|-------------|
| Unreviewed | No indicator (default) | -- |
| Approved | Green dot left of unit card, `--highlight-confirmed` bg | SQLite |
| Rejected | Red dot left of unit card, `--highlight-rejected` bg, 50% opacity text | SQLite |
| Edited | Blue dot left of unit card + "edited" label | SQLite (stores original + edited text) |
| Proposed new class | Orange badge on tree node + unit card header | Pipeline output |
| Proposed class approved | Green badge replaces orange | SQLite |
| Proposed class rejected | Red strikethrough on badge | SQLite |

## Accessibility Contract

| Requirement | Implementation |
|-------------|---------------|
| Focus management | Visible focus rings (2px `--accent` outline, 2px offset) on all interactive elements |
| Screen reader | `aria-label` on all icon-only buttons; `role="tree"` + `role="treeitem"` on FOLIO tree; `aria-expanded` on collapsible nodes |
| Color contrast | All text meets WCAG AA (4.5:1 minimum). `--text` (#e4e6f0) on `--bg` (#0f1117) = 13.2:1. `--text-dim` (#8b8fa3) on `--surface` (#1a1d27) = 4.8:1. |
| Keyboard navigation | All actions reachable via keyboard (see shortcuts table above) |
| Reduced motion | Respect `prefers-reduced-motion: reduce` -- disable all transitions/animations |
| Status announcements | `aria-live="polite"` region for review action confirmations ("Unit approved", "Unit rejected") |
