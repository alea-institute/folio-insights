    - `def format_units_json(self, units: list[KnowledgeUnit], corpus: CorpusManifest) -> dict`:
      Returns a dict structured for JSON serialization:
      ```python
      {
          "corpus": corpus.name,
          "generated_at": iso_timestamp,
          "total_units": len(units),
          "by_confidence": {"high": count, "medium": count, "low": count},
          "by_type": {"advice": count, "principle": count, ...},
          "units": [unit.model_dump() for unit in units],  # Full data
      ```

    **1. Create `src/folio_insights/pipeline/orchestrator.py`:**

    `class PipelineOrchestrator`:
    - `def __init__(self, settings: Settings)`: stores settings, initializes stage list and checkpoint manager
    - Stage execution order (7 stages):
      1. IngestionStage
      2. StructureParserStage
      3. BoundaryDetectionStage
      4. DistillerStage
      5. KnowledgeClassifierStage
      6. FolioTaggerStage
      7. DeduplicatorStage
    - `async def run(self, source_dir: Path, corpus_name: str | None = None) -> InsightsJob`:
      1. Create InsightsJob with source_dir and corpus_name
      2. Load corpus registry from `settings.output_dir / corpus_name / "registry.json"`
      3. For each stage in order:
         a. Check if checkpoint exists: `{output_dir}/{corpus_name}/checkpoints/{stage.name}.json`
         b. If checkpoint exists and is valid (hash matches), load and skip stage
         c. Otherwise, run `await stage.execute(job)`
         d. Save checkpoint after successful stage execution: serialize job.units and job.metadata to JSON
         e. Log stage name, duration, and unit count
      4. After all stages: run ConfidenceGate, OutputFormatter, write output files
      5. Update corpus registry, save to disk
      6. Return completed job

    `class PipelineCheckpoint`:
    - `def save(self, stage_name: str, job: InsightsJob, output_dir: Path)`: serialize to `{output_dir}/checkpoints/{stage_name}.json`
    - `def load(self, stage_name: str, output_dir: Path) -> InsightsJob | None`: load if exists
    - `def has_checkpoint(self, stage_name: str, output_dir: Path) -> bool`: check file existence
    - `def invalidate(self, stage_name: str, output_dir: Path)`: delete checkpoint file
    - Checkpoint format: `{"stage": name, "timestamp": iso, "unit_count": N, "job": job.model_dump()}`

    **2. Create `src/folio_insights/cli.py`:**

    Using Click (per CONTEXT.md decision):

    `@click.group()` -- main CLI group "folio-insights"

    `@click.command("extract")`:
    - Arguments: `source_dir` (required, path to directory of source files)
    - Options:
      - `--corpus` / `-c`: corpus name (default: "default")
      - `--output` / `-o`: output directory (default: "./output")
      - `--confidence-high`: high confidence threshold (default: 0.8)
      - `--confidence-medium`: medium confidence threshold (default: 0.5)
      - `--resume / --no-resume`: resume from last checkpoint (default: True)
      - `--verbose / -v`: verbose output
    - Implementation:
      1. Validate source_dir exists and contains files
      2. Create Settings from CLI options + env vars
      3. Create PipelineOrchestrator(settings)
      4. Run `asyncio.run(orchestrator.run(source_dir, corpus_name))`
      5. Print summary: files processed, units extracted, confidence breakdown, output location
      6. Exit 0 on success, exit 1 on error with clear message

    `@click.command("serve")`:
    - Starts the FastAPI review viewer server (placeholder for Plan 01-04)
    - Options: `--port` (default: calculated by hash), `--host` (default: "127.0.0.1")
