Build the interactive review viewer: FastAPI backend with SQLite review persistence and SvelteKit frontend with three-pane layout, FOLIO concept tree, knowledge unit detail view, source context, and full review workflow with keyboard shortcuts.

Output: Working review viewer accessible via `folio-insights serve` that loads extraction output and provides the full review workflow.

    **1. Create `api/db/models.py`:**
    SQLite schema (created via aiosqlite, no ORM):
    ```sql
    CREATE TABLE IF NOT EXISTS review_decisions (
        id INTEGER PRIMARY KEY AUTOINCREMENT,
        unit_id TEXT NOT NULL UNIQUE,
        corpus_name TEXT NOT NULL,
        status TEXT NOT NULL DEFAULT 'unreviewed',  -- unreviewed/approved/rejected/edited
        edited_text TEXT,
        original_text TEXT,  -- stored so edits can be compared/reverted
        reviewer_note TEXT DEFAULT '',
        reviewed_at TEXT,
        created_at TEXT NOT NULL DEFAULT (datetime('now')),
        updated_at TEXT NOT NULL DEFAULT (datetime('now'))
    );

    CREATE TABLE IF NOT EXISTS proposed_class_decisions (
        id INTEGER PRIMARY KEY AUTOINCREMENT,
        concept_label TEXT NOT NULL,
        corpus_name TEXT NOT NULL,
        status TEXT NOT NULL DEFAULT 'pending',  -- pending/approved/rejected
        reviewer_note TEXT DEFAULT '',
        reviewed_at TEXT,
        UNIQUE(concept_label, corpus_name)
    );

    CREATE INDEX IF NOT EXISTS idx_review_corpus ON review_decisions(corpus_name);
    CREATE INDEX IF NOT EXISTS idx_review_status ON review_decisions(status);
    ```

    **2. Create `api/db/session.py`:**
    - `async def get_db(corpus_name: str) -> aiosqlite.Connection`: opens SQLite DB at `{output_dir}/{corpus_name}/review.db`, creates tables if not exist
    - `async def init_db(db_path: Path)`: create tables using schema from models.py
    - Connection management: open per-request, close after. aiosqlite.connect with `isolation_level=None` for autocommit.

    **3. Create `api/main.py`:**
    - FastAPI app: `title="folio-insights Review Viewer"`, CORS enabled for dev (allow localhost:5173)
    - Mount API routes at `/api/v1/`
    - Mount SvelteKit build at `/` via StaticFiles (if build dir exists, otherwise skip)
    - Startup event: load extraction data from `{output_dir}/{corpus_name}/extraction.json` into memory
    - Dependency injection: provide extraction data and db connection to route handlers

    **4. Create `api/routes/tree.py`:**
    - `GET /api/v1/tree?corpus={name}`: returns FOLIO concept tree filtered to show only concepts that have tagged knowledge units
      - Build tree from extraction data: group units by folio_tags[*].iri, build parent/child hierarchy using FolioService
      - Each node: `{iri, label, children: [...], unit_count: int, branch: str}`
      - Filter: only include concepts that have at least 1 tagged unit
    - `GET /api/v1/tree/flat?corpus={name}`: returns flat list of concepts with unit counts (for list view mode)

    **5. Create `api/routes/review.py`:**
    - `GET /api/v1/units?corpus={name}&concept_iri={iri}&confidence={band}`:
      - Returns knowledge units tagged with given concept, optionally filtered by confidence band
      - Each unit includes its current review status from SQLite (joined query)
      - Response: list of units with review_status field merged in
    - `POST /api/v1/units/{unit_id}/review`:
      - Body: `{status: "approved"|"rejected"|"edited", edited_text?: string, note?: string}`
      - Upserts review_decisions row in SQLite
      - Returns updated unit with review status
    - `POST /api/v1/units/bulk-approve`:
      - Body: `{unit_ids: list[str]}` or `{confidence_min: float}` (approve all above threshold)
      - Batch upsert approved status for all matching units
    - `GET /api/v1/review/stats?corpus={name}`:
      - Returns: `{total, approved, rejected, edited, unreviewed, by_confidence: {high, medium, low}}`
    - `POST /api/v1/proposed-classes/{label}/review`:
      - Body: `{status: "approved"|"rejected", note?: string}`
      - Upserts proposed_class_decisions
    - `POST /api/v1/review/reset?corpus={name}`:
      - Deletes all review decisions for corpus (destructive, used for re-review)

    **6. Create `api/routes/source.py`:**
    - `GET /api/v1/source?file={path}&start={int}&end={int}`:
      - Reads source file from disk, returns text context around the span
      - Returns: `{text: str, file_path: str, section_breadcrumb: str, found: bool}`
      - If file not found: returns `{found: false, message: "Source file not available"}`
      - Context window: 500 chars before start, span text, 500 chars after end
      - NEVER persists or caches source text -- reads from disk each time

    - `test_review_edit`: POST with status="edited" and edited_text -> verify text is stored and original preserved
    - `test_review_persist`: approve unit, restart app (new test client), GET unit -> verify still approved (SQLite persistence)
    - `test_bulk_approve`: POST /api/v1/units/bulk-approve with confidence_min=0.8 -> verify all high-confidence units approved
    - `test_review_stats`: approve some, reject some, GET /api/v1/review/stats -> verify counts match
