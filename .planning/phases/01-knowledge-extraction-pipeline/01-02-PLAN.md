<objective>
Build the core extraction pipeline: tiered boundary detection, idea distillation, knowledge type classification with novelty scoring, four-path FOLIO concept tagging with 5-stage confidence scoring, and cross-document deduplication.

Purpose: This is the heart of folio-insights -- transforming raw ingested text into typed, FOLIO-tagged knowledge units with full provenance. The tiered boundary detection (structural -> embedding -> LLM) minimizes expensive LLM calls while the four-path FOLIO extraction maximizes tagging recall against the full 18K-concept ontology.

Output: Complete extraction pipeline stages that take ingested/structured text and produce fully tagged, classified, deduplicated KnowledgeUnit objects.
</objective>

    **1. Create `src/folio_insights/services/boundary/structural.py`:**

    `def detect_structural_boundaries(elements: list[StructuredElement]) -> list[Boundary]`:
    - `class Boundary(BaseModel)`: start: int, end: int, source_file: str, text: str, section_path: list[str], confidence: float, method: str
    - Rules (Tier 1, handles ~70-80% of cases, FREE):
      - Heading change = definite boundary (confidence 1.0)
      - Each bullet/numbered list item = separate boundary (confidence 0.9)
      - Double newline (paragraph break) = candidate boundary (confidence 0.7)
      - Transition words at paragraph start ("However,", "In contrast,", "Alternatively,", "Note that", "Warning:", "Tip:", "Important:") = candidate boundary (confidence 0.8)
    - Output: list of Boundary objects, one per detected unit
    - Each boundary carries its section_path from the StructuredElement

    **2. Create `src/folio_insights/services/boundary/semantic.py`:**

    `def detect_semantic_boundaries(sentences: list[str], model_name: str = "all-MiniLM-L6-v2", threshold: float = 0.3) -> list[int]`:
    - Tier 2 (handles ~15-20%, CPU-only, cheap):
    - Load sentence-transformers model (cache the model instance at module level for reuse)
    - Encode all sentences into embeddings
    - Compute cosine similarity between consecutive sentence pairs
    - Where similarity drops below `threshold` (default 0.3), mark as a boundary index
    - Return list of sentence indices where boundaries occur
    - Use for paragraphs that Tier 1 did NOT split (i.e., long paragraphs without structural markers)

    **3. Create `src/folio_insights/services/boundary/llm_refiner.py`:**

    `async def refine_boundaries_with_llm(text: str, candidate_boundaries: list[Boundary], llm_bridge: LLMBridge) -> list[Boundary]`:
    - Tier 3 (handles ~5%, expensive, only for ambiguity):
    - Only called for text segments where:
      - Tier 1 produced a single boundary for >500 chars of text (likely multiple ideas lumped together)
      - Tier 2 produced boundaries that split mid-sentence (possibly wrong)
    - Use instructor for structured output: send the ambiguous text segment to LLM with prompt from `prompts/boundary.py`
    - LLM returns a list of `BoundaryRefinement(BaseModel)` objects: start_char, end_char, rationale
    - Temperature = 0 for extraction consistency
    - Each refined boundary gets confidence based on LLM agreement with structural cues

    `class HeadingContextExtractor`:
    - The fourth extraction path per CONTEXT.md: document structure -> FOLIO concept mapping
    - `async def extract_heading_concepts(section_path: list[str], folio_service) -> list[ConceptTag]`:
      1. For each heading level in section_path (from most specific to least):
         - Search FOLIO via `folio_service.search_by_label(heading_text)`
         - If top match confidence >= 0.7: create ConceptTag with extraction_path="heading_context", confidence weighted by proximity:
           - Immediate subheading (last in section_path): weight 1.0
           - Parent heading (second-to-last): weight 0.7
           - Chapter title (first in section_path): weight 0.4
         - If confidence < 0.7: reduce weight further, fall back to parent headings
      2. Return list of ConceptTags from heading context (may be empty if no headings map well to FOLIO)

    `class FourPathReconciler`:
    - Extends folio-enrich's Reconciler from 2-path to 4-path
    - `def __init__(self, base_reconciler)`: wraps the imported Reconciler
    - `def reconcile(self, ruler_concepts, llm_concepts, semantic_concepts, heading_concepts) -> list[ReconciledConcept]`:
      1. Run base 2-path reconciliation: `base_reconciler.reconcile_with_embedding_triage(ruler_concepts, llm_concepts)` -- this is proven, don't touch it
      2. Integrate semantic path: for each semantic concept, if it matches a base result -> boost confidence by 0.1; if new -> add with extraction_path="semantic"
      3. Integrate heading context: heading concepts never override base results; they only boost confidence of matching concepts OR add new concept suggestions with extraction_path="heading_context"
      4. Each final concept records which paths contributed (for display in review viewer)
    - `class ReconciledConcept(BaseModel)`: iri, label, confidence, contributing_paths (list[str]), branch
