## Accomplishments

- Processing store manages full EventSource lifecycle with typed SSE events (status, activity, complete) and clean state machine transitions

- ActivityLog provides collapsible monospace timestamped entries with auto-scroll and chevron toggle
- UploadZone supports drag-and-drop, click-to-browse, and folder upload (webkitdirectory) for all 14 formats plus ZIP
- All components use Svelte 5 runes ($state, $derived, $effect, $props) and follow established project patterns
- Full ARIA accessibility: role="button" on upload zone, role="progressbar", aria-expanded on log, aria-live region for screen reader announcements

## Decisions Made
- Processing store uses module-level EventSource variable with explicit lifecycle functions rather than store-based cleanup
- UploadZone has separate hidden inputs for file picker and folder upload (webkitdirectory attribute)
- FileList uses text badges (e.g., "TXT", "PDF") for format display rather than SVG icons -- simpler and more readable
- ProgressDisplay derives all stage states from comparing currentStage position in the ordered STAGES array
- Auto-navigation uses $effect watching processingStatus with 1.5s setTimeout delay before goto()
- Upload page renders four conditional states: no corpus selected, idle (upload zone + file list), processing (progress + log), complete (success message)
