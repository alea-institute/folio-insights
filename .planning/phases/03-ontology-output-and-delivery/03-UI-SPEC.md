## Component Inventory

### New Components

#### 1. ExportDialog.svelte

**Purpose:** Modal dialog for selecting export formats and triggering OWL export. Follows ManualTaskDialog pattern (overlay + dialog + focus trap).

**Layout:**
```
+--------------------------------------+
|  Export Ontology              [x]    |
|                                      |
|  Export approved tasks and knowledge |
|  units as FOLIO-compatible files.    |
|                                      |
|  OUTPUT FORMATS                      |
|  [x] OWL/XML (.owl)                 |
|  [x] Turtle (.ttl)                  |
|  [ ] JSON-LD (.jsonld)              |
|  [ ] HTML browsable site            |
|  [ ] Markdown outline               |
|                                      |
|  OPTIONS                             |
|  [x] Run SHACL validation           |
|  [x] Generate changelog             |
|                                      |
|  --- validation summary (post-run) --+
|  SHACL: PASS  |  RDF: PASS          |
|  IRIs: 42 new | 0 collisions        |
|  -----------------------------------+
|                                      |
|            Cancel    Export Ontology  |
+--------------------------------------+
```

**Dimensions:**
- Dialog max-width: 480px (wider than ManualTaskDialog's 440px to accommodate checkbox descriptions)
- Width: 90% of viewport (responsive, same as existing dialogs)
- Padding: `var(--lg)` (24px) — matches existing dialog pattern
- Border-radius: 8px — matches existing dialog pattern
- Overlay: `rgba(0, 0, 0, 0.5)` — matches existing overlay pattern
- z-index: 100 — matches existing dialog z-index

**States:**
1. **Idle** — checkboxes selectable, "Export Ontology" button enabled when at least one format checked
2. **Exporting** — button shows "Exporting..." with reduced opacity (0.5), checkboxes disabled, no close on overlay click
3. **Complete** — validation summary section appears below options, button changes to "Download" with download icon, individual format download links appear
4. **Error** — error message in `var(--red)` below the options section, button reverts to "Retry Export"

**Checkbox styling:**
- Size: 16x16px custom checkbox (not native)
- Unchecked: `var(--surface2)` background, `var(--border)` border, 4px border-radius
- Checked: `var(--accent)` background, white checkmark SVG (12x12 stroke-width 2)
- Label: 13px, `var(--text)`, 8px gap from checkbox
- Description (optional per format): 11px, `var(--text-dim)`, indented under label

**Focus trap:** Manual Tab/Shift+Tab interception matching ConfirmDialog and ManualTaskDialog pattern.

**Keyboard:**
- Escape: close dialog (unless exporting)
- Enter: trigger export (when idle) or download (when complete)
- Space: toggle focused checkbox

## Interaction Contracts

### Export Flow

1. User clicks "Export" button in tree header (or presses `x` keyboard shortcut)
2. ExportDialog opens with OWL/XML and Turtle pre-checked (these are the FOLIO maintainer essentials), SHACL validation pre-checked, changelog pre-checked
3. User toggles additional formats as desired
4. User clicks "Export Ontology" or presses Enter
5. Button enters exporting state; checkboxes become disabled; overlay click does not close dialog
6. On completion: ValidationSummary appears with pass/warn/fail badges; button changes to "Download Files"; individual format links appear below checkboxes (each format that was selected gets a download link)
7. User clicks "Download Files" to download a ZIP of all selected formats, or clicks individual format links
8. Dialog closes on Escape or Cancel (not during export)

### Download Mechanism

- Single format: browser downloads the file directly via `<a download>` pointing to `/api/v1/corpus/{id}/export/{format}` endpoint
- Multiple formats: trigger a ZIP download via `/api/v1/corpus/{id}/export/bundle?formats=owl,ttl,jsonld` endpoint
- File naming: `folio-insights-{corpus_name}.{ext}` for individual files, `folio-insights-{corpus_name}-export.zip` for bundle

### Keyboard Shortcuts (additions to existing KeyboardShortcuts.svelte)

| Key | Context | Action |
|-----|---------|--------|
| `x` | Tasks page, not in dialog | Open Export Dialog |
| `Escape` | Export Dialog open, not exporting | Close Export Dialog |
| `Enter` | Export Dialog open, idle | Trigger export |
| `Enter` | Export Dialog open, complete | Download files |
| `Space` | Checkbox focused | Toggle checkbox |

### Accessibility

- Dialog uses `role="dialog"` (not `alertdialog` — export is not destructive) with `aria-modal="true"`
- `aria-labelledby` points to dialog title, `aria-describedby` points to description paragraph
- Checkboxes use native `<input type="checkbox">` with `<label>` association (not custom divs with role="checkbox")
- ValidationSummary badges use `aria-label` for screen reader context (e.g., `aria-label="SHACL validation: passed"`)
- Export progress announced via `aria-live="polite"` region (reuse existing `.sr-only` pattern from Tasks page)
- Focus returns to Export button in tree header when dialog closes
