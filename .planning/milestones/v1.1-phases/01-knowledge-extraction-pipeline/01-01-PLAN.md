    **1. Create `src/folio_insights/pipeline/stages/ingestion.py`:**

    `class IngestionStage(InsightsPipelineStage)`:
    - `name` property returns `"ingestion"`
    - `async execute(job: InsightsJob) -> InsightsJob`:
      1. Walk `job.source_dir` recursively, collecting files matching supported extensions: `.md`, `.txt`, `.docx`, `.pdf`, `.html`, `.htm`, `.rtf`, `.eml`, `.msg`, `.xml`, `.csv`, `.xlsx`, `.tsv`, `.wpd`
      2. For each file, check `corpus_registry.needs_processing(file_path)` -- skip if already processed
      3. Route to appropriate ingestor:
         - `.md`, `.txt`, `.docx`, `.pdf`, `.html`, `.htm`, `.rtf`, `.eml`, `.msg` -> `IngestionBridge.detect_and_ingest()`
         - `.csv`, `.xlsx`, `.tsv` -> `MapperBridge.parse_tabular()`
         - `.xml` -> new XML ingestor using lxml: `lxml.etree.parse()` -> extract text content from all text nodes, preserve element hierarchy as section_path
         - `.wpd` -> if `settings.doctor_url` is set, POST to `{doctor_url}/api/v1/convert/wpd/to/html/` then parse HTML result via IngestionBridge; if not set, log warning and skip
      4. For each ingested file, create `CorpusDocument` with file_path, content_hash (SHA-256), format name, unit_count=0 (populated later)
      5. Append all documents to `job.documents`
      6. Store raw text and TextElements on `job.metadata["ingested"]` as a dict keyed by file_path: `{"text": str, "elements": list[dict]}`
      7. Record lineage on job metadata: stage="ingestion", action="ingest_directory", detail=f"{len(new_files)} files ingested"

    **2. Create `src/folio_insights/pipeline/stages/structure_parser.py`:**

    `class StructureParserStage(InsightsPipelineStage)`:
    - `name` property returns `"structure_parser"`
    - `async execute(job: InsightsJob) -> InsightsJob`:
      1. For each ingested document in `job.metadata["ingested"]`:
         - Extract heading hierarchy from TextElements: build a tree of headings (h1 > h2 > h3 > ...) tracking `section_path: list[str]` at each depth
         - For each paragraph/list-item element, attach the current `section_path` as context (e.g., `["Chapter 8", "Expert Witnesses", "Methodology Challenges"]`)
         - Store structured output on `job.metadata["structured"]` keyed by file_path: list of `StructuredElement` dicts with fields: `text`, `element_type` (heading/paragraph/list_item), `section_path`, `level`, `char_offset_start`, `char_offset_end`
      2. Record lineage

    - element_type: str (one of "heading", "paragraph", "list_item")
    - section_path: list[str]
    - level: int
