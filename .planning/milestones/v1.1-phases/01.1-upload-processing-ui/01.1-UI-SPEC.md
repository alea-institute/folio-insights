## Spacing Scale

Declared values (must be multiples of 4) -- inherited from existing `app.css`:

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px (`--xs`) | Icon gaps, inline padding, stage pill gaps, activity log entry padding |
| sm | 8px (`--sm`) | Compact element spacing, card padding, file list item gaps, pill padding |
| md | 16px (`--md`) | Default element spacing, upload zone padding, section padding |
| lg | 24px (`--lg`) | Header horizontal padding, empty state padding |
| xl | 32px (`--xl`) | Layout gaps between sidebar and main content |
| 2xl | 48px (`--2xl`) | Upload zone vertical padding (drag-drop target area) |

Exceptions:
- Upload zone minimum height: 120px (ensures comfortable drag target)
- File list item height: 36px (consistent with tree node height of 28px + padding)
- Touch target minimum: 44px for the "Process" button and corpus action buttons

## Typography

All values match established project conventions from `app.css` and existing components.

| Role | Size | Weight | Line Height | Usage in This Phase |
|------|------|--------|-------------|---------------------|
| Body | 14px | 400 | 1.5 | File names, corpus descriptions, activity log messages |
| Label | 13px | 400 | 1.5 | Corpus list items, file metadata, upload zone hint text |
| Small | 11px | 400 | 1.4 | Timestamps in activity log, file sizes, status text, monospace log entries, stage pill text |
| Heading | 18px | 600 | 1.2 | Page section titles ("Upload Documents", corpus name heading) |

Additional typographic rules:
- Monospace for activity log: `'SF Mono', 'Fira Code', 'Consolas', monospace` at 11px (matches folio-enrich activity log)
- Stage pill text: 11px uppercase with `letter-spacing: 0.5px` (Small role with transform, replaces prior 10px)
- Letter-spacing on headings: `-0.3px` (matches existing `.concept-label` and `.app-title`)
- Corpus sidebar item name: 13px at weight 400 (Label role), distinguished from subtitle via `color: var(--text)` vs `var(--text-dim)`

## Interaction States

### Upload Flow

1. **Empty corpus** -- Upload zone centered, no file list visible, Process button disabled
2. **Files uploaded** -- Upload zone at top, file list below with pending indicators, Process button enabled
3. **Processing** -- Progress display replaces upload zone, Process button shows "Processing..." (disabled), file list items update status in real-time
4. **Complete** -- Brief success state (progress bar green, all pills green) then auto-navigate to Review page after 1.5s delay
5. **Error** -- Progress bar turns red at failure point, error stage pill highlighted, activity log auto-expands to show error details, Process button re-enables as "Retry Processing"

### Drag-and-Drop

- `dragenter`/`dragover`: Upload zone border changes to `var(--accent)`, background gains subtle blue tint
- `dragleave`: Revert to default state
- `drop`: Files added to file list, upload zone reverts. Brief file-added animation (fade-in on new list items, 150ms)
- During processing: drag events ignored, upload zone not visible

### SSE Connection States

| State | User Feedback |
|-------|---------------|
| Connected | Activity log streaming, stages advancing |
| Reconnecting | Status text: "Reconnecting..." in `var(--orange)` |
| Failed | Status text: "Connection lost" in `var(--red)`, "Retry" link |

### Page Navigation

- Header tabs: Upload / Review
- Corpus selector in header shared across both pages
- Changing corpus on Upload page loads that corpus's files
- Auto-navigate to Review after processing uses SvelteKit `goto('/review')` or equivalent route transition

## Accessibility

- Upload zone: `role="button"`, `tabindex="0"`, `aria-label="Upload files. Drop files here or press Enter to open file picker."`
- File input: visually hidden (`display: none`), triggered by upload zone click
- Corpus list: `role="listbox"`, each item `role="option"`, `aria-selected`
- File list: `role="list"`, each item `role="listitem"`
- Progress bar: `role="progressbar"`, `aria-valuenow`, `aria-valuemin="0"`, `aria-valuemax="100"`, `aria-label="Pipeline processing progress"`
- Stage pills: `aria-current="step"` on active stage
- Activity log: `aria-expanded` on toggle, `aria-controls` linking to log container
- Delete confirmation: `role="alertdialog"`, `aria-modal="true"`, focus trapped within dialog
- Navigation tabs: `role="tablist"` / `role="tab"` / `aria-selected` (matches existing header tab pattern)
- Status announcements: Use existing `aria-live="polite"` region for processing updates ("Processing stage: {stage_name}", "Processing complete")
- Focus visible: `2px solid var(--accent)`, `outline-offset: 2px` (already in `app.css`)
- Keyboard: Enter/Space to activate upload zone, Escape to close dialogs
