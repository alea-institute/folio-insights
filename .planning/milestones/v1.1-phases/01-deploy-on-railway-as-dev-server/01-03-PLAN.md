<interfaces>
**Railway CLI reference:** https://docs.railway.com/guides/cli

**Key commands (for the human-action task):**
```bash
# One-time (per machine)
npm i -g @railway/cli              # or: brew install railway
railway login                       # opens browser for OAuth — REQUIRES USER

# One-time (per project)
cd /home/damienriehl/Coding\ Projects/folio-insights
railway init                        # creates a new Railway project, prompts for name
railway link                        # if joining existing project

# Connect git for auto-redeploy
railway connect                     # link the Railway service to this GitHub repo + branch
# Or set via dashboard: Service -> Settings -> Source -> connect GitHub repo, branch=master

# Deploy
railway up                          # uploads build context, builds via Dockerfile, deploys
# OR just `git push` once git is connected

# Inspect
railway status                      # shows current deployment
railway logs                         # tail live logs
railway domain                       # prints the assigned *.up.railway.app URL
railway open                         # opens dashboard
```

**Troubleshooting the deploy (if healthcheck fails):**
- Check `railway logs` for uvicorn startup errors (most likely: missing fastapi/uvicorn — fix by adding to pyproject.toml)
- Check that Railway assigned a PORT (it does automatically, via $PORT env var)
- Confirm /health works against the built image locally first: `docker run -p 8000:8000 <image> && curl localhost:8000/health`

    ## Deploying to Railway

    folio-insights ships as a single-service Railway deployment (FastAPI + bundled
    SvelteKit viewer + bundled `output/` data, all in one container). Dev
    environment only; production hardening is out of scope.

    **Live dev URL:** {LIVE_URL}

    **Prerequisites:**

    - Railway account and the `railway` CLI (`npm i -g @railway/cli` or
      `brew install railway`)
    - This repo checked out locally

    **First-time deploy:**

    ```bash
    cd folio-insights
    railway login                 # browser OAuth
    railway init                  # create a new Railway project
    railway up                    # build the Dockerfile and deploy
    railway domain                # show the *.up.railway.app URL
    ```

    Then connect the service to this GitHub repo via the Railway dashboard
    (Service → Settings → Source → Connect GitHub, branch `master`) so pushes
    auto-redeploy.

    **How it builds:**

    - [`Dockerfile`](Dockerfile) — multi-stage: Node 20 builds the SvelteKit
      viewer (adapter-static), Python 3.11-slim installs the backend with `uv`,
      copies the built viewer, and bundles `output/` into the image.
    - [`railway.toml`](railway.toml) — pins the Dockerfile builder and
      `/health` healthcheck path; restart policy is `ON_FAILURE` with 3 retries.
    - FastAPI serves the viewer as static files at `/` and routes API calls at
      `/api/*` — single origin, no CORS in prod.

    **What this deploy does NOT include** (deferred to later milestones):

    - Custom domain / TLS termination
    - Authentication / access control
    - Postgres migration (SQLite only)
    - Staging/prod separation
    - CI checks before deploy
    ```
