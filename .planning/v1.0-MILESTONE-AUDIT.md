## Integration Breaks (Blocking)

### Break 1: ExportDialog permanently disabled

**File:** `viewer/src/routes/tasks/+page.svelte` line 42
**Code:** `hasApprovedTasks` checks `n.review_status === 'complete'` and `n.is_task`
**Issue:** Task tree API returns `"approved"` (not `"complete"`) and never emits `is_task` field
**Effect:** ExportDialog always shows "No approved tasks" — Export button permanently suppressed
**Fix:** Change to `n.review_status === 'approved'` and add `is_task` to API node shape

### Break 2: triggerExport parses ZIP as JSON

**File:** `viewer/src/lib/api/client.ts` lines 520-530
**Code:** `triggerExport` calls `request<T>()` which runs `res.json()` on ZIP binary
**Issue:** Bundle endpoint returns `application/zip` bytes — JSON parse throws SyntaxError
**Effect:** Every export attempt via dialog reports "Export failed" even on success
**Fix:** Use `fetch` directly for bundle endpoint, handle binary response separately

### Break 3: TaskTreeNode schema mismatch (non-blocking)

**File:** `viewer/src/lib/api/client.ts` line 261
**Issue:** `path` and `sortOrder` declared but not returned by API
**Effect:** Undefined at runtime; tree renders but LTree path-based sorting is lost

### Break 4: CLI serve stub (non-blocking)

**File:** `src/folio_insights/cli.py` lines 498-518
**Issue:** `folio-insights serve` prints placeholder instead of starting FastAPI
**Effect:** Users cannot launch review viewer via documented CLI command
